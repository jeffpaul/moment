# Project Moment: Visual Mockup Prompts

Use these prompts with an image generation or design tool to create cleaner mockups.

## Prompt 1: Mobile Publishing Flow

Create a clean product mockup board for a mobile-first WordPress publishing experience called Project Moment.

Show six iPhone-style screens in a horizontal flow:

1. Moment Home with a large New Moment button and recent drafts.
2. Camera Roll picker showing photos and videos.
3. Compose Moment screen with selected media, caption field, optional AI Assist button, and clear Publish button.
4. AI Assist bottom sheet with suggested caption, alt text, and tags, with Accept, Edit, and Ignore actions.
5. Publish screen showing destination as Your Site required and optional syndication toggles preselected based on Moment type, such as image to Instagram, text to Bluesky, and video to YouTube.
6. Published success screen with View on Site and Create Another actions.

The design should be minimal, phone-first, clean, and creator-friendly. It should not look like WordPress admin. Emphasize fast publishing, content ownership, and optional AI assistance. Use polished SaaS-style product mockup aesthetics with clear readable UI text.

## Prompt 2: Site Presentation Views

Create a clean product mockup board showing how Project Moment content appears on a WordPress site.

Show five mobile website screens:

1. Timeline feed with mixed text, image, video, and gallery posts.
2. Images grid view at `/images`.
3. Videos feed at `/videos`.
4. Notes stream at `/notes`.
5. Profile/home page with a social-style profile header and recent Moments.

The message should be: same content, different views, still WordPress.

The design should feel like a modern social profile but owned by the user's website. Avoid WordPress admin styling. Emphasize that the site is the source of truth.

## Prompt 3: First-Run Onboarding

Create a clean product mockup board for Project Moment onboarding.

Show five mobile screens:

1. Choose your Moment address, with example `jane.moment.site` and an option to connect a domain later.
2. Pick a starter profile style, showing three simple visual templates.
3. Optional AI Assist setup, with a Skip button and a note that AI providers can be connected later through WordPress Connectors.
4. Publish your first Moment, showing a camera roll picker.
5. Your site is live, showing the first Moment on the user's own site.

The design should communicate a very short onboarding flow for a low-friction WordPress publishing experience. The key message is: your first post is your first site.

## Prompt 4: One-Slide Concept Diagram

Create a simple concept diagram for Project Moment.

Left side: Social Apps Today.

Flow:
Camera Roll → Instagram / TikTok / Threads / Bluesky / Mastodon / YouTube

Message: Platforms become the home of your content.

Right side: Project Moment.

Flow:
Camera Roll → Your WordPress Site → Instagram / TikTok / Threads / Bluesky / Mastodon / YouTube

Message: Your site is the source of truth. Social networks are distribution.

Make it clean, presentation-ready, and easy to understand in five seconds.

## Prompt 5: Product Hero Image

Create a polished hero image for a product concept called Project Moment: a new phone-first Personal Site Publisher Mode for WordPress.

## Prompt 6: Default Sharing Rules

Create a clean mobile product mockup board for Project Moment showing default sharing rules by Moment type.

Show four phone screens:

1. Sharing Defaults settings screen with rows:
   - Text Moments → Bluesky
   - Image Moments → Instagram
   - Video Moments → YouTube
   - Gallery Moments → Instagram
   - Mixed Media → Ask each time

2. Connected Destinations screen showing Your Site always connected, Bluesky connected, Instagram connected, YouTube connected, Mastodon available, and TikTok / Threads / X as future or disconnected.

3. Publish Image Moment screen showing Your Site required and Instagram preselected automatically, with Bluesky and Mastodon available but off.

4. Publish Text Moment screen showing Your Site required and Bluesky preselected automatically, with Mastodon available but off.

The design should be minimal, mobile-first, creator-friendly, and not look like WordPress admin. The main message is: Moment publishes to your site first and remembers where each kind of Moment should go next.


## Mockup Prompt: Home Screen Launch

Create a clean mobile product mockup showing Project Moment as an app-like web experience. Show a smartphone home screen with a simple Moment app icon, then a second phone screen where Moment opens directly to a mobile-first publishing home at `/moment`. The UI should feel like a lightweight app, with a large New Moment button, recent moments, and no WordPress admin navigation. Include subtle labeling that this can be launched from a phone home screen as a PWA or browser shortcut. Keep the design minimal, modern, and focused on quick publishing.


## Prompt 5: Moment Notifications

Create a clean mobile product mockup board for a Moment notifications screen.

Show four iPhone-style screens:

1. Notifications list showing replies and comments grouped by original Moment, with clear source labels like `Reply from Bluesky`, `Comment from Instagram`, `Comment from YouTube`, and `On-site comment`.
2. Notification detail screen showing the original Moment at the top and a combined thread of on-site comments plus imported social responses below it.
3. Filter sheet with options for All, On-site, Bluesky, Instagram, YouTube, and Mastodon.
4. Empty state saying: “Replies and comments on your Moments will appear here.” Include a small note that normal WordPress post comments are hidden from Moment notifications by default.

The design should feel like a simple social app inbox, not WordPress comment moderation. Keep it phone-first, calm, readable, and creator-friendly. Emphasize that WordPress is the place where the conversation comes back together.
